3.1.1.2 Use Case Scenario #2 <Mengelola Profil>

Skenario Use Case #2:

i. Pre-Condition
  • User sudah login.
  • User memiliki akun yang valid.
ii. Use Case Description
a. Primary Flow (Melihat Profil)
  • User mengakses halaman profil melalui menu navigasi atau avatar.
  • Informasi profil user ditampilkan (Full Name, Username, Role).
  • User dapat melihat foto profil dan cover photo.
b. Primary Flow (Mengedit Profil)
  • User mengklik tombol "Edit" pada field yang ingin diubah.
  • Form edit untuk field tersebut ditampilkan.
  • User mengisi data baru.
  • User menyimpan perubahan.
  • Data divalidasi dan disimpan ke database.
  • Notifikasi sukses ditampilkan.
c. Primary Flow (Mengubah Password)
  • User mengklik tombol "Edit" pada field Password.
  • Form untuk mengubah password ditampilkan.
  • User mengisi password lama, password baru, dan konfirmasi password baru.
  • Password lama divalidasi dengan data di database.
  • Jika valid, password baru disimpan.
  • Notifikasi sukses ditampilkan.
d. Alternative Flow
  • User mencoba mengubah username dengan format yang tidak valid.
  • Pesan error ditampilkan (username harus 3-30 karakter, hanya huruf, angka, dan underscore).
  • User memperbaiki input dan mencoba lagi.
iii. Post-Condition
  • Data profil user berhasil diperbarui dalam database.
  • Perubahan langsung terlihat di halaman profil.
