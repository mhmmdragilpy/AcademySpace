3.2.1.5 Use Case Scenario <Extend: Melihat Jadwal Reservasi> ADMIN EXTEND

Relasi: Extend dari Use Case "FR-14: Melihat Jadwal Reservasi"
Deskripsi: Use case ini memperluas fungsionalitas melihat jadwal reservasi dengan menambahkan tampilan kalender interaktif (Calendar View) untuk memvisualisasikan jadwal reservasi secara lebih intuitif.

Skenario Use Case Extend - Calendar View:

i. Pre-Condition
  • Admin sudah login dengan role Admin.
  • Admin sedang berada di halaman "Manage Reservations".
  • Terdapat data reservasi dalam database.
  • Tab "Calendar View" tersedia di halaman.
ii. Extension Point
  • Extend dari FR-14 pada titik: "Admin ingin melihat jadwal dalam format kalender".
  • Kondisi aktivasi: Admin mengklik tab "Calendar View".
iii. Use Case Description
a. Primary Flow (Menampilkan Calendar View)
  • Admin mengklik tab "Calendar View" pada halaman Manage Reservations.
  • Sistem memuat data reservasi untuk bulan dan tahun saat ini.
  • Kalender bulanan ditampilkan dengan grid 7 kolom (Minggu-Sabtu).
  • Setiap sel tanggal menampilkan indikator jumlah reservasi.
  • Reservasi ditampilkan dengan warna berbeda berdasarkan status:
    • Pending: Kuning/Abu-abu terang
    • Approved: Hijau/Hitam solid
    • Rejected: Merah/Garis diagonal
    • Cancelled: Abu-abu/Transparan
  • Legend warna status ditampilkan di bawah kalender.
b. Primary Flow (Navigasi Bulan)
  • Admin melihat bulan dan tahun aktif di header kalender.
  • Admin mengklik tombol "←" untuk bulan sebelumnya atau "→" untuk bulan berikutnya.
  • Sistem memuat ulang data reservasi untuk bulan yang dipilih.
  • Tampilan kalender diperbarui dengan data baru.
  • Admin dapat kembali ke bulan saat ini dengan tombol "Today".
c. Primary Flow (Melihat Detail Tanggal)
  • Admin mengklik salah satu sel tanggal di kalender.
  • Popup modal "Day Details" muncul.
  • Modal menampilkan daftar reservasi pada tanggal tersebut:
    • Nama fasilitas
    • Waktu reservasi (jam mulai - jam selesai)
    • Nama pengguna yang melakukan reservasi
    • Status reservasi
    • Tombol aksi: View Detail, Approve/Reject (jika pending)
  • Admin dapat menutup popup dengan tombol close atau klik di luar modal.
d. Primary Flow (Filter Berdasarkan Fasilitas)
  • Admin mengklik dropdown "Filter by Facility".
  • Daftar semua fasilitas ditampilkan.
  • Admin memilih satu fasilitas.
  • Kalender diperbarui menampilkan hanya reservasi untuk fasilitas yang dipilih.
  • Admin dapat memilih "All Facilities" untuk menampilkan semua reservasi kembali.
e. Primary Flow (Filter Berdasarkan Status)
  • Admin mengklik dropdown "Filter by Status".
  • Pilihan status: All, Pending, Approved, Rejected, Cancelled.
  • Admin memilih status tertentu.
  • Kalender diperbarui menampilkan hanya reservasi dengan status yang dipilih.
f. Alternative Flow (Tidak Ada Reservasi)
  • Jika tidak ada reservasi pada bulan yang dipilih.
  • Kalender tetap ditampilkan dengan sel tanggal kosong (tanpa indikator).
  • Pesan "No reservations found for this month" ditampilkan di bawah kalender.
g. Alternative Flow (Error Loading Data)
  • Jika terjadi error saat memuat data reservasi.
  • Pesan error ditampilkan: "Failed to load calendar data".
  • Tombol "Retry" tersedia untuk memuat ulang data.
iv. Post-Condition
  • Admin dapat melihat jadwal reservasi dalam format kalender bulanan.
  • Admin dapat mengidentifikasi tanggal-tanggal dengan reservasi banyak (busy days).
  • Admin dapat melihat distribusi status reservasi secara visual.
  • Admin dapat dengan cepat mengakses detail reservasi pada tanggal tertentu.
  • Admin dapat merencanakan maintenance fasilitas berdasarkan kepadatan jadwal.
