3.2.1.4 Use Case Scenario #16 <Melihat Jadwal Reservasi> ADMIN

Skenario Use Case #16:

i. Pre-Condition
  • Admin sudah login dengan role Admin.
  • Terdapat data reservasi dalam database.
ii. Use Case Description
a. Primary Flow (List View)
  • Admin mengakses menu "Reservations" dari sidebar.
  • Halaman Manage Reservations ditampilkan.
  • Admin melihat tab "List View" yang aktif secara default.
  • Semua reservasi ditampilkan dalam bentuk tabel.
  • Admin dapat melihat jadwal reservasi berdasarkan tanggal dan waktu.
b. Primary Flow (Calendar View)
  • Admin mengklik tab "Calendar View".
  • Kalender dengan reservasi yang ditandai ditampilkan.
  • Admin dapat navigasi bulan untuk melihat jadwal reservasi.
  • Reservasi ditampilkan dengan warna berbeda berdasarkan status:
    • Pending: Kuning/Abu-abu
    • Approved: Hijau/Hitam
    • Rejected: Merah/Garis
  • Admin dapat mengklik tanggal untuk melihat detail reservasi pada tanggal tersebut.
c. Primary Flow (Filter Jadwal)
  • Admin menggunakan filter status untuk melihat jadwal tertentu.
  • Admin dapat filter berdasarkan fasilitas tertentu.
  • Tampilan diperbarui sesuai filter.
d. Alternative Flow
  • Tidak ada reservasi pada bulan yang dipilih.
  • Kalender menampilkan tanggal tanpa penanda reservasi.
iii. Post-Condition
  • Admin dapat melihat jadwal reservasi secara keseluruhan.
  • Admin dapat mengidentifikasi slot waktu yang padat atau kosong.
  • Admin dapat merencanakan maintenance fasilitas berdasarkan jadwal.
