3.1.1.5 Use Case Scenario #5 <Menerima Notifikasi Approval/Rejection>

Skenario Use Case #5:

i. Pre-Condition
  • User sudah login.
  • User telah mengajukan minimal satu reservasi.
  • Admin telah memproses (approve/reject) reservasi user.
ii. Use Case Description
a. Primary Flow (Melihat Notifikasi)
  • User mengklik ikon notifikasi (Bell Icon) pada navigation bar.
  • Halaman notifikasi ditampilkan.
  • Daftar notifikasi ditampilkan dengan informasi:
    • Judul notifikasi
    • Deskripsi singkat
    • Waktu notifikasi diterima
    • Status (read/unread)
  • Notifikasi terbaru ditampilkan di bagian atas.
b. Primary Flow (Notifikasi Approval)
  • Admin menyetujui reservasi user.
  • Notifikasi baru dibuat untuk user.
  • Badge notifikasi pada ikon Bell bertambah.
  • User membuka halaman notifikasi dan melihat pesan "Reservasi Anda telah disetujui".
  • User dapat mengklik notifikasi untuk melihat detail reservasi.
c. Primary Flow (Notifikasi Rejection)
  • Admin menolak reservasi user.
  • Notifikasi baru dibuat untuk user dengan alasan penolakan.
  • User membuka halaman notifikasi dan melihat pesan "Reservasi Anda ditolak".
  • User dapat melihat alasan penolakan dan mengajukan reservasi ulang.
d. Alternative Flow
  • User tidak memiliki notifikasi baru.
  • Pesan "Tidak ada notifikasi" ditampilkan.
iii. Post-Condition
  • User mengetahui status reservasi mereka (Approved/Rejected).
  • Notifikasi yang sudah dibaca ditandai sebagai "read".
  • User dapat mengambil tindakan lanjutan berdasarkan status.
