3.2.1.3 Use Case Scenario #15 <Melihat Detail Reservasi> ADMIN

Skenario Use Case #15:

i. Pre-Condition
  • Admin sudah login dengan role Admin.
  • Terdapat minimal satu reservasi dalam database.
ii. Use Case Description
a. Primary Flow
  • Admin mengakses menu "Reservations" dari sidebar.
  • Tabel daftar semua reservasi ditampilkan.
  • Admin dapat melihat informasi singkat setiap reservasi:
    • Nama dan username user
    • Fasilitas yang direservasi
    • Tanggal dan waktu (mulai - selesai)
    • Status reservasi
b. Primary Flow (Melihat Dokumen Pendukung)
  • Admin melihat kolom "Document" pada tabel.
  • Jika ada dokumen, tombol "View" ditampilkan.
  • Admin mengklik tombol "View".
  • Dokumen pendukung (Proposal/Rundown) dibuka di tab baru.
  • Admin dapat mengunduh atau mencetak dokumen jika diperlukan.
c. Primary Flow (Filter berdasarkan Status)
  • Admin menggunakan dropdown filter "All Status".
  • Admin memilih status tertentu (Pending/Approved/Rejected).
  • Reservasi difilter dan ditampilkan sesuai status.
d. Alternative Flow
  • Reservasi tidak memiliki dokumen pendukung.
  • Kolom Document menampilkan "No Doc" atau kosong.
  • Admin tetap dapat melakukan approval berdasarkan informasi lain.
iii. Post-Condition
  • Admin dapat melihat semua informasi detail reservasi.
  • Admin dapat mengambil keputusan untuk approve/reject. (Include: FR-09)
