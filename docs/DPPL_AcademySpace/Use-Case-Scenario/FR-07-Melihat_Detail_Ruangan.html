3.1.1.7 Use Case Scenario #7 <Melihat Detail Ruangan>

Skenario Use Case #7:

i. Pre-Condition
  • User telah mencari atau memilih fasilitas dari halaman Home atau Check Availability.
  • Fasilitas yang dipilih ada dalam database.
ii. Use Case Description
a. Primary Flow
  • User mengklik tombol "Book Now" atau nama fasilitas dari daftar.
  • Halaman detail ruangan ditampilkan dengan informasi:
    • Nama Ruangan
    • Gedung/Lokasi
    • Kapasitas
    • Deskripsi lengkap fasilitas
    • Rating dan ulasan
  • Kalender ketersediaan bulanan ditampilkan.
  • User dapat navigasi bulan untuk melihat ketersediaan.
  • User memilih tanggal pada kalender.
  • Slot waktu yang tersedia dan terpesan ditampilkan.
b. Primary Flow (Lanjut ke Booking)
  • User melihat form "Ajukan Pemesanan Digital".
  • User melihat panduan Smart Booking Process (Upload Proposal → Smart Approval → Get E-Permit).
  • User mengisi form booking dan submit. (Include: FR-03 Mengajukan Reservasi)
c. Alternative Flow
  • Fasilitas dalam status "Maintenance" atau "Inactive".
  • Informasi bahwa fasilitas tidak tersedia untuk booking ditampilkan.
  • User dapat mencari fasilitas lain.
iii. Post-Condition
  • User mendapatkan informasi lengkap tentang ruangan.
  • User mengetahui ketersediaan ruangan pada tanggal tertentu.
  • User dapat melanjutkan ke proses reservasi.
