3.1.1.3 Use Case Scenario #3 <Mengajukan Reservasi>

Skenario Use Case #3:

i. Pre-Condition
  • User sudah login.
  • User telah memilih fasilitas yang ingin direservasi.
  • Fasilitas dalam status "Active" (tersedia untuk booking).
ii. Use Case Description
a. Primary Flow
  • User mengakses halaman detail ruangan. (Include: FR-07 Melihat Detail Ruangan)
  • User memilih tanggal reservasi dari kalender ketersediaan.
  • Slot waktu yang tersedia untuk tanggal tersebut ditampilkan.
  • User memilih waktu mulai dan waktu selesai.
  • User mengisi jumlah peserta.
  • User mengunggah dokumen pendukung (Proposal/Rundown) jika diperlukan.
  • User mengisi keperluan penggunaan ruangan.
  • User mengklik tombol "Ajukan Pemesanan".
  • Data reservasi divalidasi.
  • Reservasi disimpan dengan status "Pending".
  • User diarahkan ke halaman My Reservations.
b. Alternative Flow (Slot Tidak Tersedia)
  • User memilih tanggal dari kalender.
  • Semua slot pada tanggal tersebut sudah terpesan.
  • User memilih tanggal lain yang tersedia.
c. Alternative Flow (Validasi Gagal)
  • User mengisi form reservasi dengan data yang tidak lengkap.
  • Pesan error ditampilkan untuk field yang wajib diisi.
  • User melengkapi data dan mencoba submit ulang.
iii. Post-Condition
  • Reservasi tersimpan dalam database dengan status "Pending".
  • Admin menerima notifikasi untuk mereview reservasi.
  • User dapat melihat status reservasi di halaman My Reservations.
