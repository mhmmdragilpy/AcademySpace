3.1.1.6 Use Case Scenario #6 <Mencari Fasilitas>

Skenario Use Case #6:

i. Pre-Condition
  • User mengakses halaman utama (Home) Academy Space.
  • Terdapat data fasilitas yang terdaftar dalam database.
ii. Use Case Description
a. Primary Flow (Pencarian via Search Bar)
  • User mengakses halaman Home.
  • User melihat hero section dengan search bar.
  • User mengetik kata kunci pencarian (nama fasilitas, gedung, dll).
  • Fasilitas yang sesuai dengan kata kunci ditampilkan.
  • User dapat melihat hasil pencarian dalam bentuk grid card.
b. Primary Flow (Browsing Fasilitas)
  • User scroll ke bagian "Explore Our Facilities".
  • Daftar fasilitas ditampilkan dalam bentuk card.
  • Setiap card menampilkan:
    • Gambar fasilitas
    • Status (Active/Maintenance/Inactive)
    • Nama gedung
    • Nama fasilitas
    • Kapasitas dan fitur
    • Tombol "Book Now"
  • User mengklik "Book Now" untuk melanjutkan ke detail ruangan. (Include: FR-07 Melihat Detail Ruangan)
c. Alternative Flow
  • User mencari dengan kata kunci yang tidak ditemukan.
  • Pesan "Tidak ada fasilitas yang sesuai" ditampilkan.
  • User dapat mengubah kata kunci atau menggunakan filter. (Extend: FR-12 Menyaring Pencarian Fasilitas)
iii. Post-Condition
  • User menemukan fasilitas yang dicari.
  • User dapat melanjutkan ke proses melihat detail atau booking.
