3.1.1.1 Use Case Scenario #1 <Membuat atau Masuk Akun>

Skenario Use Case #1:

i. Pre-Condition
  • Akun user belum tervalidasi oleh data.
  • User memiliki akses ke halaman utama website Academy Space.
ii. Use Case Description
a. Primary Flow (Register)
  • User mengakses halaman register melalui tombol "Sign Up".
  • User memilih register sebagai Mahasiswa (User) atau Admin.
  • User mengisi biodata yang diperlukan (Full Name, Username, Password, Confirm Password).
  • Jika memilih Admin, user mengisi Admin Registration Token yang valid.
  • Data user dimasukkan ke dalam basis data.
  • User diarahkan ke halaman login.
b. Primary Flow (Login)
  • User mengakses halaman login melalui tombol "Log in".
  • User memilih login sebagai User atau Admin.
  • User mengisi username dan password.
  • Data user divalidasi sesuai dengan basis data.
  • Jika valid, user diarahkan ke halaman utama (My Reservations untuk User, Admin Dashboard untuk Admin).
c. Alternative Flow
  • User mengakses halaman login.
  • User mengisi username dan password.
  • Data user tidak sesuai dengan basis data sehingga tidak valid.
  • Pesan error ditampilkan.
  • User mengakses halaman lupa password. (Extend: EXT-Mereset_Password)
iii. Post-Condition
  • Akun user tervalidasi oleh data.
  • User dapat mengakses fitur-fitur sesuai dengan role (User/Admin).
  • Session login tersimpan di browser.
