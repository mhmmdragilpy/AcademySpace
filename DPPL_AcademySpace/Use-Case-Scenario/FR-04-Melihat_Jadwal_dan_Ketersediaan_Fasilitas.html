3.1.1.4 Use Case Scenario #4 <Melihat Jadwal dan Ketersediaan Fasilitas>

Skenario Use Case #4:

i. Pre-Condition
  • User mengakses website Academy Space.
  • Terdapat data fasilitas yang sudah terdaftar dalam database.
ii. Use Case Description
a. Primary Flow
  • User mengakses halaman "Check Availability" dari menu navigasi atau footer.
  • Daftar semua fasilitas yang tersedia ditampilkan.
  • User dapat menggunakan search bar untuk mencari fasilitas berdasarkan nama.
  • User dapat memfilter fasilitas berdasarkan tipe (Auditorium, Lab, Meeting Room, dll).
  • Informasi singkat setiap fasilitas ditampilkan (nama, gedung, kapasitas, status).
  • User memilih fasilitas untuk melihat detail ketersediaan.
b. Primary Flow (Melihat Kalender Ketersediaan)
  • User masuk ke halaman detail fasilitas.
  • Kalender ketersediaan bulanan ditampilkan.
  • User dapat navigasi ke bulan sebelumnya atau berikutnya.
  • User memilih tanggal tertentu pada kalender.
  • Slot waktu yang tersedia dan terpesan untuk tanggal tersebut ditampilkan.
c. Alternative Flow
  • User mencari fasilitas dengan kata kunci yang tidak ditemukan.
  • Pesan "Tidak ada fasilitas yang ditemukan" ditampilkan.
  • User mengubah kata kunci pencarian atau menghapus filter.
iii. Post-Condition
  • User mendapatkan informasi ketersediaan fasilitas secara real-time.
  • User dapat melanjutkan ke proses reservasi jika slot tersedia.
