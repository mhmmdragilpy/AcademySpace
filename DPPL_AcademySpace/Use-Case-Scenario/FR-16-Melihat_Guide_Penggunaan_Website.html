3.1.1.11 Use Case Scenario #11 <Melihat Guide Penggunaan Website>

Skenario Use Case #11:

i. Pre-Condition
  • User mengakses website Academy Space.
  • User ingin memahami cara menggunakan fitur reservasi.
ii. Use Case Description
a. Primary Flow
  • User mengklik menu "Guide" pada navigation bar atau footer.
  • Halaman "Digital Booking Guide" ditampilkan.
  • User membaca panduan yang mencakup:
    • Langkah 1: Akses menu 'Availability' untuk melihat ketersediaan ruangan secara real-time dan transparan.
    • Langkah 2: Lakukan reservasi sepenuhnya secara online. Isi formulir digital dan unggah dokumen pendukung (Proposal/Rundown) tanpa perlu mencetak berkas fisik.
    • Langkah 3: 'Smart Approval' mempercepat proses verifikasi. Pantau status pengajuan (Pending/Approved) langsung dari Dashboard atau Notifikasi.
    • Langkah 4: Ketentuan Waktu - Pengajuan kegiatan reguler minimal H-3, event besar minimal H-7.
    • Langkah 5: Setelah disetujui, E-Permit diterbitkan otomatis. Tunjukkan E-Permit kepada petugas di lokasi.
    • Langkah 6: Layanan terintegrasi 24/7. Tidak perlu datang ke kantor fisik.
b. Primary Flow (Navigasi dari Guide)
  • User mengklik tombol "Cek Ketersediaan" untuk langsung ke halaman Check Availability.
  • User mengklik tombol "Riwayat Reservasi" untuk melihat reservasi yang sudah dibuat.
c. Alternative Flow
  • User belum login tetapi ingin membuat reservasi.
  • User diarahkan ke halaman login sebelum dapat membuat reservasi.
iii. Post-Condition
  • User memahami alur penggunaan fitur reservasi.
  • User mengetahui ketentuan dan syarat untuk melakukan reservasi.
  • User siap untuk menggunakan fitur-fitur Academy Space.
