3.1.1.12 Use Case Scenario <Mereset Password> EXTEND

Skenario Use Case (Extension dari FR-01):

i. Pre-Condition
  • User memiliki akun yang terdaftar dalam database.
  • User lupa password dan tidak dapat login.
  • User berada di halaman login dan mengklik "Forgot password?".
ii. Use Case Description
a. Primary Flow
  • User mengklik link "Forgot password?" di halaman login.
  • Halaman forgot password ditampilkan.
  • User melihat informasi:
    • Judul "Forgot Password"
    • Deskripsi alur reset password
  • User mengisi username yang terdaftar.
  • User mengisi Password Reset Master Token (didapat dari Admin).
  • User mengisi password baru.
  • User mengisi konfirmasi password baru.
  • User mengklik tombol "Reset Password".
  • Token dan username divalidasi dengan data di database.
  • Jika valid, password user diupdate.
  • Pesan sukses dan link ke halaman login ditampilkan.
b. Alternative Flow (Token Invalid)
  • User mengisi token yang salah atau expired.
  • Pesan error "Token tidak valid" ditampilkan.
  • User harus menghubungi Admin untuk mendapatkan token yang benar.
c. Alternative Flow (Username Tidak Ditemukan)
  • User mengisi username yang tidak terdaftar.
  • Pesan error "Username tidak ditemukan" ditampilkan.
  • User memeriksa kembali username atau membuat akun baru.
d. Alternative Flow (Password Tidak Cocok)
  • User mengisi password baru dan konfirmasi yang tidak sama.
  • Pesan error "Password tidak cocok" ditampilkan.
  • User memperbaiki input password.
iii. Post-Condition
  • Password user berhasil direset.
  • User dapat login dengan password baru.
  • Password lama tidak lagi valid.
