3.1.1.9 Use Case Scenario #9 <Melihat Riwayat Reservasi>

Skenario Use Case #9:

i. Pre-Condition
  • User sudah login.
  • User memiliki minimal satu reservasi dalam database.
ii. Use Case Description
a. Primary Flow
  • User mengakses halaman "My Reservations" melalui menu navigasi.
  • Daftar semua reservasi user ditampilkan dalam bentuk tabel.
  • Setiap baris reservasi menampilkan:
    • Nama fasilitas
    • Tanggal dan waktu reservasi
    • Status (Pending/Approved/Rejected/Cancelled/Completed)
    • Aksi yang tersedia (Edit/Cancel)
  • Reservasi diurutkan berdasarkan tanggal terbaru.
b. Primary Flow (Filter dan Pencarian)
  • User dapat memfilter reservasi berdasarkan status.
  • User dapat mencari reservasi berdasarkan nama fasilitas.
  • Tampilan tabel diperbarui sesuai filter.
c. Primary Flow (Aksi pada Reservasi)
  • User mengklik "Edit" untuk mengubah reservasi. (Include: FR-08 Mengedit Reservasi)
  • User mengklik "Cancel" untuk membatalkan reservasi. (Include: FR-08 Membatalkan Reservasi)
d. Alternative Flow
  • User tidak memiliki reservasi.
  • Pesan "Anda belum memiliki reservasi" ditampilkan.
  • Tombol untuk membuat reservasi baru disediakan.
iii. Post-Condition
  • User dapat melihat semua riwayat reservasi mereka.
  • User mengetahui status terkini dari setiap reservasi.
  • User dapat mengambil tindakan pada reservasi yang masih aktif.
