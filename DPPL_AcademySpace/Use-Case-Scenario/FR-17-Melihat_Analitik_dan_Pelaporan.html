3.2.1.6 Use Case Scenario #18 <Melihat Analitik dan Pelaporan> ADMIN

Skenario Use Case #18:

i. Pre-Condition
  • Admin sudah login dengan role Admin.
  • Terdapat data historis reservasi dalam database.
ii. Use Case Description
a. Primary Flow (Melihat Statistics Overview)
  • Admin mengakses menu "Analytics" dari sidebar.
  • Halaman Analytics & Reporting ditampilkan.
  • Admin melihat kartu statistik ringkas:
    • Total Reservations: Jumlah total semua reservasi
    • Approved: Jumlah reservasi yang disetujui
    • Pending: Jumlah reservasi yang menunggu approval
    • Past 30 Days: Jumlah reservasi dalam 30 hari terakhir
b. Primary Flow (Melihat Chart Visualisasi)
  • Admin scroll ke bagian chart.
  • Berbagai visualisasi data ditampilkan:
    • Reservation Status Distribution: Pie chart proporsi status reservasi
    • Top 5 Facilities: Bar chart fasilitas paling banyak direservasi
    • Daily Reservation Trend: Line chart tren reservasi harian
    • Weekly Reservation Activity: Bar chart aktivitas per hari dalam seminggu
c. Primary Flow (Melihat Tabel Utilisasi Fasilitas)
  • Admin melihat tabel "Facility Utilization".
  • Tabel menampilkan untuk setiap fasilitas:
    • Nama fasilitas
    • Tipe
    • Gedung
    • Kapasitas
    • Total reservasi
    • Rata-rata peserta
  • Admin dapat mengidentifikasi fasilitas dengan utilisasi tinggi/rendah.
d. Primary Flow (Melihat Top Active Users)
  • Admin melihat tabel "Top Active Users".
  • Tabel menampilkan user dengan reservasi terbanyak:
    • Nama user
    • Username
    • Department/Fakultas
    • Total reservasi
e. Alternative Flow
  • Tidak ada data reservasi dalam periode tertentu.
  • Chart dan statistik menampilkan nilai 0 atau kosong.
  • Admin dapat memilih periode waktu berbeda.
iii. Post-Condition
  • Admin mendapatkan insight tentang penggunaan fasilitas.
  • Admin dapat membuat keputusan berdasarkan data (menambah/mengurangi fasilitas, dll).
  • Data dapat digunakan untuk laporan kepada manajemen.
