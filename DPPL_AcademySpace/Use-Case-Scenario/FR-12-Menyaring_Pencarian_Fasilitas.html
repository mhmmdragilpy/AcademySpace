3.1.1.10 Use Case Scenario #10 <Menyaring Pencarian Fasilitas>

Skenario Use Case #10:

i. Pre-Condition
  • User mengakses halaman Home atau Check Availability.
  • Terdapat daftar fasilitas yang ditampilkan.
ii. Use Case Description
a. Primary Flow (Filter berdasarkan Tipe)
  • User melihat daftar fasilitas pada halaman Home.
  • User melihat tombol filter di atas grid fasilitas (All, Classrooms, Auditoriums, Labs, dll).
  • User mengklik salah satu tombol filter.
  • Hanya fasilitas dengan tipe yang dipilih ditampilkan.
  • Tombol filter yang aktif ditandai dengan style berbeda (primary button).
b. Primary Flow (Reset Filter)
  • User mengklik tombol "All".
  • Semua fasilitas ditampilkan tanpa filter.
c. Primary Flow (Kombinasi Filter dan Pencarian)
  • User menggunakan search bar untuk mencari dengan kata kunci.
  • User kemudian mengklik filter tipe.
  • Fasilitas yang memenuhi kedua kriteria (kata kunci DAN tipe) ditampilkan.
d. Alternative Flow
  • Tidak ada fasilitas yang sesuai dengan filter yang dipilih.
  • Pesan "Tidak ada fasilitas untuk kategori ini" ditampilkan.
  • User dapat memilih filter lain atau reset ke "All".
iii. Post-Condition
  • Daftar fasilitas berhasil difilter sesuai kriteria.
  • User dapat lebih mudah menemukan fasilitas yang diinginkan.
