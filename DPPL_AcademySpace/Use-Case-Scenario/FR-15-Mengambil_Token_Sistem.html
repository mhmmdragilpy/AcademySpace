3.2.1.5 Use Case Scenario #17 <Mengambil Token> ADMIN

Skenario Use Case #17:

i. Pre-Condition
  • Admin sudah login dengan role Admin.
  • Admin memiliki akses ke menu Tokens.
ii. Use Case Description
a. Primary Flow (Melihat Token)
  • Admin mengakses menu "System Tokens" dari sidebar.
  • Halaman Tokens ditampilkan dengan dua kartu:
    • Admin Registration Token: Token yang dibutuhkan untuk mendaftarkan akun Admin baru.
    • Password Reset Master Token: Token yang digunakan untuk validasi reset password tanpa email.
  • Setiap kartu menampilkan token string yang tersembunyi/terlihat.
b. Primary Flow (Menyalin Token)
  • Admin mengklik tombol "Copy Token" pada kartu token yang diinginkan.
  • Token disalin ke clipboard.
  • Notifikasi "Token copied to clipboard" ditampilkan.
  • Admin dapat membagikan token kepada user yang membutuhkan.
c. Use Case: Memberikan Admin Registration Token
  • Admin menyalin Admin Registration Token.
  • Admin memberikan token kepada calon Admin baru melalui channel aman (tatap muka, email internal, dll).
  • Calon Admin menggunakan token saat registrasi sebagai Admin.
d. Use Case: Memberikan Password Reset Token
  • User melaporkan lupa password dan meminta reset.
  • Admin menyalin Password Reset Master Token.
  • Admin memberikan token kepada user yang terverifikasi identitasnya.
  • User menggunakan token di halaman Forgot Password.
⚠️ Peringatan Keamanan: Token bersifat sensitif. Pastikan hanya membagikan kepada pihak yang berwenang dan melalui channel yang aman.
iii. Post-Condition
  • Token berhasil disalin ke clipboard.
  • Admin dapat menggunakan token untuk keperluan yang sesuai.
  • Keamanan data tetap terjaga jika token dijaga kerahasiaannya.
