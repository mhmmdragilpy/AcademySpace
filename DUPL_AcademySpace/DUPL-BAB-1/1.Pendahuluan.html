BAB 1: Pendahuluan

1.1 Tujuan Pembuatan Dokumen

Dokumen Uji Perangkat Lunak (DUPL) ini disusun sebagai panduan komprehensif untuk melakukan pengujian terhadap sistem Academy Space, sebuah aplikasi berbasis web untuk reservasi fasilitas kampus di lingkungan Universitas Telkom. Dokumen ini bertujuan untuk:

  1. Memberikan panduan sistematis dalam melaksanakan pengujian perangkat lunak Academy Space.
  2. Mendefinisikan ruang lingkup, strategi, dan metode pengujian yang akan diterapkan.
  3. Menjadi acuan bagi tim pengembang dan penguji dalam memastikan kualitas perangkat lunak sebelum diimplementasikan.
  4. Mendokumentasikan hasil pengujian sebagai bukti verifikasi dan validasi sistem.
  5. Mengidentifikasi defect atau kegagalan sistem agar dapat diperbaiki sebelum deployment.

Dokumen ini diperuntukkan bagi tim pengembang, tim Quality Assurance (QA), dosen pembimbing, serta pemangku kepentingan lainnya yang terlibat dalam proyek Academy Space.

1.2 Ruang Lingkup Pengujian

Pengujian yang dilakukan dalam dokumen ini mencakup seluruh fungsionalitas utama sistem Academy Space yang telah didefinisikan dalam Spesifikasi Kebutuhan Perangkat Lunak (SKPL) dan Deskripsi Perancangan Perangkat Lunak (DPPL). Ruang lingkup pengujian meliputi:

  • Pengujian Unit: Pengujian pada level method dan class untuk memastikan setiap komponen berfungsi dengan benar secara independen.
  • Pengujian Integrasi: Pengujian interaksi antar komponen dan modul sistem untuk memvalidasi alur proses bisnis.
  • Pengujian Fungsional: Pengujian berdasarkan use case yang telah didefinisikan untuk memastikan sistem memenuhi kebutuhan pengguna.
  • User Acceptance Test (UAT): Pengujian oleh pengguna potensial untuk memvalidasi kesesuaian sistem dengan ekspektasi pengguna.
Catatan: Pengujian tidak mencakup pengujian performa (load testing) dan pengujian keamanan (security testing) secara mendalam, karena proyek ini bersifat akademis dan pembelajaran.

1.4 Overview Sistem dan Fitur Utama

Academy Space adalah sistem reservasi fasilitas kampus berbasis web yang dirancang untuk memudahkan mahasiswa dan admin dalam mengelola peminjaman ruangan di lingkungan Universitas Telkom. Sistem ini dibangun menggunakan teknologi modern dengan arsitektur client-server.

1.4.1 Deskripsi Sistem

Sistem terdiri dari dua jenis pengguna utama: User (Mahasiswa) dan Admin. User dapat melakukan pencarian fasilitas, melihat ketersediaan jadwal, mengajukan reservasi, dan memantau status reservasi. Admin bertugas mengelola fasilitas, menyetujui atau menolak reservasi, serta melihat analitik penggunaan fasilitas.

1.5.5 Strategi dan Metode Pengujian

Pengujian perangkat lunak Academy Space menggunakan strategi dan metode berikut:

A. Strategi Pengujian

  • Bottom-Up Testing: Pengujian dimulai dari level unit (method/function), kemudian berlanjut ke pengujian integrasi antar modul.
  • Risk-Based Testing: Prioritas pengujian diberikan pada modul dengan tingkat risiko dan dampak tertinggi terhadap sistem.

B. Metode Pengujian

  1. White Box Testing:
    • Pengujian berdasarkan struktur internal kode
    • Menggunakan teknik basis path testing dengan cyclomatic complexity
    • Memastikan setiap path dalam kode teruji
  2. Black Box Testing:
    • Pengujian berdasarkan spesifikasi fungsional
    • Menggunakan teknik equivalence partitioning dan boundary value analysis
    • Fokus pada input-output tanpa melihat kode internal
  3. Automated Testing:
    • Menggunakan Jest untuk pengujian unit otomatis
    • Test cases dapat dijalankan berulang kali secara konsisten
