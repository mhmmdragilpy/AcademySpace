#7 Mengajukan Reservasi

Deskripsi: User mengajukan reservasi fasilitas untuk kegiatan tertentu.

Aktor: User (Mahasiswa/Dosen)

Pre-condition

  • User telah login kedalam sistem.
  • Fasilitas tersedia pada tanggal yang dipilih.

Primary Flow

  1. User memilih fasilitas dan melihat kalender.
  2. Sistem menampilkan slot waktu yang tersedia (Hijau) dan terisi (Merah).
  3. User memilih slot waktu yang diinginkan.
  4. User mengisi form: Jumlah Peserta, Keperluan, Upload Proposal (PDF).
  5. User menekan tombol "Ajukan Reservasi".
  6. Sistem memvalidasi input (kelengkapan data dan bentrok jadwal).
  7. Sistem menyimpan data reservasi dengan status "PENDING".
  8. Sistem menampilkan notifikasi sukses dan redirect ke halaman Riwayat.

Post-condition

  • Data reservasi tersimpan di database.
  • Slot waktu tercatat sementara agar tidak dibooking orang lain (walaupun masih pending).

#9 Menyetujui atau Menolak Reservasi

Deskripsi: Admin memproses pengajuan reservasi yang masuk.

Aktor: Admin

Primary Flow

  1. Admin login dan membuka halaman "Manage Reservations".
  2. Sistem menampilkan daftar reservasi dengan status "PENDING".
  3. Admin memilih salah satu reservasi untuk melihat detail (Proposal, Keperluan).
  4. Admin menekan tombol "Approve" (Setuju) atau "Reject" (Tolak).
  5. Jika Reject, Admin wajib mengisi alasan penolakan.
  6. Sistem mengupdate status reservasi di database.
  7. Sistem mengirim notifikasi ke User.

#4 Mencari Fasilitas

Deskripsi: Sistem menyediakan fitur pencarian fasilitas.

Aktor: User/Guest

Primary Flow

  1. User mengakses halaman utama atau halaman eksplorasi.
  2. User mengetik kata kunci pada search bar (e.g. "Aula").
  3. Sistem melakukan query ke database menggunakan `ILIKE` atau pencarian teks.
  4. Sistem menampilkan daftar fasilitas yang relevan.
